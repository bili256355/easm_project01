V6_1 structure manifest
======================

scripts/run_stage_partition_v6_1.py
  Independent entrypoint for the V6_1 child version.

src/stage_partition_v6_1/window_layer_config.py
  V6_1-specific configuration. Reads V6 outputs through an explicit source config.

src/stage_partition_v6_1/window_band_builder.py
  Builds candidate support bands from V6 points and the baseline detector profile.
  Inspired by V2/V3 point->band logic only.

src/stage_partition_v6_1/window_registry.py
  Merges overlapping/adjacent bands into lightweight derived windows and builds
  point-window membership provenance.

src/stage_partition_v6_1/window_uncertainty.py
  Computes return-day uncertainty for each window main peak using V6 bootstrap
  match records. Uncertainty is a window attribute, not a second window system.

src/stage_partition_v6_1/window_layer_report.py
  Writes V6_1 summary and run metadata.

src/stage_partition_v6_1/pipeline_window_layer.py
  Orchestrates the V6_1 child-version pipeline.

Key design constraints
----------------------
- V6 remains the upstream point-screening version.
- V6_1 consumes V6 outputs and reuses only stable V6 utility interfaces.
- V6_1 has its own config, pipeline, entrypoint, outputs and logs.
- No competition / parameter-path / final judgement are introduced here.
